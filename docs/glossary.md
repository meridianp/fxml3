# Glossary

This glossary provides definitions for technical terms, trading concepts, and FXML4-specific terminology used throughout the documentation.

## A

**API (Application Programming Interface)**
: A set of protocols and tools for building software applications. FXML4 provides a REST API for programmatic access.

**Algorithmic Trading**
: The use of computer programs to execute trades based on predefined rules and strategies.

**Alpha**
: The excess return of an investment relative to the return of a benchmark index.

**ATR (Average True Range)**
: A technical indicator that measures market volatility by decomposing the entire range of an asset price.

**Anthropic**
: The AI company that created Claude, including Claude Opus 4 used for visual Elliott Wave analysis.

## B

**Backtest**
: The process of testing a trading strategy on historical data to evaluate its performance.

**Backtesting Engine**
: FXML4's event-driven system that simulates realistic trading conditions for strategy evaluation.

**Bid-Ask Spread**
: The difference between the highest price a buyer is willing to pay (bid) and the lowest price a seller will accept (ask).

**Breakout**
: When price moves outside a defined support or resistance level with increased volume.

## C

**Claude Opus 4**
: Anthropic's advanced AI model used by FXML4 for visual chart analysis and pattern recognition.

**Candlestick Chart**
: A type of financial chart that displays the high, low, open, and closing prices of a security.

**Commission**
: The fee charged by a broker for executing trades.

**Corrective Wave**
: In Elliott Wave theory, a three-wave pattern (A-B-C) that moves against the main trend.

**Cross-Validation**
: A resampling procedure used to evaluate machine learning models on limited data samples.

## D

**Data Engineering**
: The process of collecting, storing, processing, and analyzing large volumes of market data.

**Drawdown**
: The peak-to-trough decline during a specific period for an investment or trading account.

**Docker**
: A platform for developing, shipping, and running applications in containers.

## E

**Elliott Wave Theory**
: A form of technical analysis that identifies recurring fractal wave patterns in financial markets.

**Entry Signal**
: A trading signal indicating when to open a position (buy or sell).

**Event-Driven Architecture**
: A software design pattern where the flow of the program is determined by events such as user actions or sensor outputs.

**Exit Signal**
: A trading signal indicating when to close an existing position.

## F

**FastAPI**
: A modern, fast web framework for building APIs with Python, used by FXML4's API server.

**Feature Engineering**
: The process of creating new features from raw data to improve machine learning model performance.

**Fibonacci Retracement**
: A technical analysis tool that uses horizontal lines to indicate support or resistance at key Fibonacci levels.

**Fill**
: The execution of an order to buy or sell a security.

**Forex (Foreign Exchange)**
: The global marketplace for exchanging national currencies.

## G

**Git**
: A distributed version control system for tracking changes in source code.

**Google Vertex AI**
: Google Cloud's unified AI platform used by FXML4 for distributed model training.

## H

**High-Frequency Trading (HFT)**
: Algorithmic trading characterized by high speeds, high turnover rates, and high order-to-trade ratios.

**Hybrid Analysis**
: FXML4's approach combining algorithmic pattern detection with AI visual analysis.

**Hyperparameter**
: A parameter whose value is set before the learning process begins in machine learning.

## I

**Impulse Wave**
: In Elliott Wave theory, a five-wave pattern (1-2-3-4-5) moving in the direction of the main trend.

**Interactive Brokers (IB)**
: A broker providing API access for automated trading, integrated with FXML4.

**Indicator**
: A mathematical calculation based on price and/or volume used in technical analysis.

## J

**JWT (JSON Web Token)**
: An open standard for securely transmitting information between parties, used for FXML4 authentication.

## K

**Kubernetes (K8s)**
: An open-source container orchestration platform for automating deployment, scaling, and management.

## L

**Latency**
: The delay between a trading signal and its execution.

**Leverage**
: Using borrowed capital to increase potential returns (and risks).

**Limit Order**
: An order to buy or sell at a specific price or better.

**LLM (Large Language Model)**
: AI models like Claude Opus 4 that can understand and generate human-like text.

**Long Position**
: Buying a currency pair expecting its value to increase.

## M

**Machine Learning (ML)**
: The use of algorithms and statistical models to perform tasks without explicit instructions.

**Market Order**
: An order to buy or sell immediately at the current market price.

**Maximum Drawdown**
: The maximum observed loss from a peak to a trough of a portfolio.

**Moving Average**
: A calculation to analyze data points by creating a series of averages of different subsets.

## N

**Neural Network**
: A machine learning model inspired by biological neural networks.

**Normalization**
: The process of organizing data to reduce redundancy and improve data integrity.

## O

**OHLC**
: Open, High, Low, Close - the four main data points for each time period.

**Order Book**
: A list of buy and sell orders for a specific security or financial instrument.

**Overfitting**
: When a model learns the training data too well, including noise, reducing generalization.

## P

**Paper Trading**
: Simulated trading that allows investors to practice without risking real money.

**Pattern Recognition**
: The automated identification of patterns in data, central to Elliott Wave analysis.

**Pip**
: The smallest price move in a currency exchange rate (usually 0.0001 for major pairs).

**PostgreSQL**
: An open-source relational database used by FXML4 for data storage.

**Position Sizing**
: Determining how much capital to allocate to a specific trade.

## Q

**Quantitative Analysis**
: The use of mathematical and statistical methods in finance and investment management.

## R

**Redis**
: An in-memory data structure store used by FXML4 for caching.

**REST API**
: An architectural style for building web services based on HTTP methods.

**Risk Management**
: The process of identification, analysis, and mitigation of investment risks.

**RSI (Relative Strength Index)**
: A momentum oscillator that measures the speed and magnitude of price changes.

## S

**Sharpe Ratio**
: A measure of risk-adjusted returns, calculated as (return - risk-free rate) / standard deviation.

**Short Position**
: Selling a currency pair expecting its value to decrease.

**Slippage**
: The difference between expected and actual transaction prices.

**Stop Loss**
: An order to close a position when it reaches a certain loss level.

**Support/Resistance**
: Price levels where a trend tends to stop and reverse.

## T

**Take Profit**
: An order to close a position when it reaches a certain profit level.

**Technical Analysis**
: Analysis based on past market data, primarily price and volume.

**TimescaleDB**
: A time-series database built on PostgreSQL, used for market data storage.

**Trading Signal**
: An indication to buy or sell based on analysis.

## U

**Unit Test**
: A software testing method where individual units of source code are tested.

## V

**Vertex AI**
: Google Cloud's machine learning platform used for model training.

**Visual Analysis**
: FXML4's innovative approach using AI to analyze chart images.

**Volatility**
: A statistical measure of the dispersion of returns for a security.

**Volume**
: The amount of an asset traded during a given period.

## W

**Walk-Forward Analysis**
: A method of backtesting that optimizes on past data then tests on future data.

**Wave Count**
: The identification and labeling of Elliott Wave patterns.

**WebSocket**
: A protocol providing full-duplex communication channels, used for real-time data.

**Win Rate**
: The percentage of trades that are profitable.

## X

**XGBoost**
: An optimized gradient boosting library used in FXML4's ML pipeline.

## Y

**Yield**
: The earnings generated and realized on an investment over a period.

## Z

**Z-Score**
: A statistical measurement describing a value's relationship to the mean of a group of values.
